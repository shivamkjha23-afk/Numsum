# Founder Admin Workflow

This workflow is for creating real starter content directly from the NumSum Labs admin panel. Keep drafts private until they are safe and ready to publish.

## Simple launch workflow

1. Log in with the founder account.
2. Open `/admin`.
3. Create the default MSME questionnaire template.
4. Create the governance document.
5. Create the first objective target.
6. Add one real knowledge item.
7. Add one real SOP.
8. Add one real research item.
9. Create a draft competition if needed.
10. Review submitted MSME challenges.
11. Publish only public-safe records.

## Admin modules

| Module | What it is for | When to use it | Keep private | Can be published |
|---|---|---|---|---|
| Problems | Review and structure MSME challenge submissions. | When an MSME submits a real operational problem. | MSME identity, contact details, internal notes, confidential process details. | Public-safe title, summary, category, process area, desired outcome. |
| Questionnaires | Build the onboarding questions used to understand MSME context. | Before collecting detailed MSME challenge information. | Internal scoring logic or private diagnostic notes. | General questionnaire labels only if intended for users. |
| Knowledge | Store practical industrial knowledge assets. | When the founder has a real explanatory resource or lesson. | Draft notes, private references, sensitive customer examples. | Practical explanation, public sources, generic industry relevance. |
| SOPs | Manage standard operating procedure content. | When a repeatable safe process should be documented. | Site-specific confidential steps or customer details. | Generic, approved SOP guidance. |
| Research | Track research and technology-watch items. | When research evidence or technology signals support MSME problems. | Internal evaluation notes and non-public partner details. | Public sources, safe summaries, practical relevance. |
| Pilots | Track pilot validation and impact learning. | Only after a real pilot is being planned or executed. | Participant details, unverified results, private implementation notes. | Verified outcomes approved by the MSME or partner. |
| Competitions | Create innovation challenge drafts. | When a structured problem is ready for solution participation. | Draft judging notes, private sponsor details, unpublished challenge material. | Published challenge brief, timeline, eligibility, public-safe problem context. |
| Governance | Maintain constitution, policies, and governance records. | When rules, operating principles, or approvals need documentation. | Internal deliberations unless approved. | Approved policies and public governance documents. |
| Objectives | Set annual or quarterly targets. | When the founder wants to define measurable operating priorities. | Internal target rationale or sensitive financial details. | Public-safe objectives if intended for transparency. |
| Execution | Track operating work and follow-through. | When coordinating launch and delivery tasks. | Internal task details, risks, owner notes. | High-level progress only if deliberately shared. |
| Contributions | Manage contribution rules and review cycles. | When defining non-equity contribution recognition. | Individual review notes and private disputes. | Clearly labeled non-equity contribution rules. |

## Publishing principle

If a record contains private MSME details, unverified impact, internal admin notes, or confidential process information, keep it private.
